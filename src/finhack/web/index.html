Methodology · how this page composes its numbers

The overlay metric block reads a cached validation JSON written by scripts/validate_case4_intraday_overlay.py. Each row is one trading session in which at least one earnings event from our universe reported between the prior session close and the current session open (conservative T+1 mapping; zero leakage). The aggregated signal is the signed sum of enhanced_pred_sign × enhanced_confidence over contributing events, and the session bet is QQQ open→close in that direction. QQQ tracks NDX, the same index MNQ futures track.

The next-session signal calls finhack.paper_signals.build_earnings_paper_signals for the configured horizon, then groups upcoming events by their target session (the next calendar day after each earnings release). A session is marked actionable only when the average per-event confidence on that session clears the configured floor (pre-registered at 0.80).

The TopStep risk panel is a calculator: given an account size, current balance, peak end-of-day balance, and today's P&L, it computes the trailing Maximum Loss Limit floor, the remaining buffer, and the distance to the profit target. The kill-switch is a display flag — nothing here connects to a real order entry system.

The upstream model card and backtest comparison are diagnostic surfaces: they show whether the underlying earnings-direction model beats a price-only baseline on the full validation set. If the uplift is zero, the overlay above is fighting a losing battle even if a particular slice looks good — that's why both views live on the same page.
